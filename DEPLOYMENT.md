# GPU Manager - Deployment Guide

## Architecture

Centralized management with OpenWebUI + GPU Manager API for automatic load balancing:

```
Manager Node (no GPU needed)          GPU Nodes (compute only)
┌─────────────────────┐              ┌──────────────────┐
│   OpenWebUI         │              │  GPU Node 1      │
│   + Caddy Auth      │              │  - Ollama        │
│   + GPU Manager API │◄────────────►│  - GPU           │
└─────────────────────┘              └──────────────────┘
         ▲                            ┌──────────────────┐
         │                            │  GPU Node 2      │
         │                            │  - Ollama        │
         │                            │  - GPU           │
         └───────────────────────────►└──────────────────┘
                                      ┌──────────────────┐
                                      │  GPU Node 3      │
                                      │  - Ollama        │
                                      │  - GPU           │
                                      └──────────────────┘
```

**Benefits:**
- Single access point (one login, one URL)
- Automatic GPU discovery and load balancing via GPU Manager API
- Automatic GPU lifecycle management (pause/resume)
- GPUs only run Ollama (maximum efficiency)
- Centralized model management
- Easy to scale (add more GPU nodes anytime)

## Quick Start

### 1. Deploy Manager Node with GPU Manager API

```bash
# Deploy manager with integrated GPU Manager API for automatic load balancing
uv run gpumanager deploy --manager <manager-ip> --with-api
```

This deploys:
- OpenWebUI (web interface)
- GPU Manager API (automatic GPU discovery and load balancing)
- Caddy (reverse proxy with authentication)

Access WebUI at: `http://<manager-ip>:8080`

**Login Credentials:**
- Basic Auth (Caddy):
  - Username: `gatekeeper`
  - Password: Check deployment logs or `/srv/shared/.env` on manager node
- WebUI Admin:
  - Email: `admin@gpumanager.local`
  - Password: Check deployment logs or `/srv/shared/.env` on manager node

### 2. Deploy GPU Nodes

```bash
# Auto-discover all GPU workspaces
uv run gpumanager deploy

# OR use manual IP list
uv run gpumanager deploy --ips gpu_ips.txt
```

### 3. Start Using GPUs

That's it! The GPU Manager API automatically:
- Discovers all GPU nodes
- Load balances requests across available GPUs
- Manages GPU lifecycle (pause/resume)
- Handles model routing

Just login to the WebUI and start chatting. The system automatically selects and manages GPUs for you.

## Model Management

### Pre-Configure Models (Recommended for Initial Setup)

Edit `gpu-node/.env` before deployment:

```bash
# Models to pull on deployment
OLLAMA_MODELS="llama3.1:8b-instruct-q8_0,mistral:7b,codellama:13b"
```

Then deploy:
```bash
uv run gpumanager deploy
```

All nodes will pull these models on first startup.

### Download Models via WebUI

With GPU Manager API integration:

1. Login to WebUI at `http://<manager-ip>:8080`
2. Download a model from the WebUI
3. The model is stored on the GPU node that processed the request
4. Optionally sync to all nodes:

```bash
uv run gpumanager sync-models --source <source-node-ip>
```

This copies models from the source node to all other GPU nodes.

**Note:** The GPU Manager API automatically routes requests to GPUs that have the requested model loaded, optimizing for performance.

### Manual Model Distribution

```bash
# Sync from a specific source to all nodes
uv run gpumanager sync-models --source <source-ip>

# Sync to manual IP list
uv run gpumanager sync-models --source <source-ip> --ips gpu_ips.txt
```

## Firewall Management

### Open Multiple Ports

```bash
# Open multiple ports at once (recommended)
uv run gpumanager open-port --ip <node-ip> --ports 8000,8080

# Open single port
uv run gpumanager open-port --ip <node-ip> --port 8080

# By workspace name
uv run gpumanager open-port --name "LLM-manager" --ports 8000,8080
```

**Default ports:**
- Manager: 8080 (WebUI)
- GPU nodes: 11434 (Ollama API)
- API Gateway: 8000 (if running the GPU manager server)

## Common Workflows

### Adding a New GPU Node

```bash
# Deploy to new node
uv run gpumanager deploy --ips new_node_ip.txt

# Sync models from existing node
uv run gpumanager sync-models --source <existing-node-ip>

# The GPU Manager API will automatically discover the new node
# No manual configuration needed!
```

### Updating Models Across All Nodes

```bash
# Download model to one node via WebUI
# Then sync to all others
uv run gpumanager sync-models --source <node-with-new-model>
```

### Redeploying

```bash
# Redeploy manager (wipes WebUI data, creates fresh admin)
uv run gpumanager deploy --manager <manager-ip>

# Redeploy GPU nodes (preserves Ollama models)
uv run gpumanager deploy
```

## Troubleshooting

### Manager Node

**Can't access WebUI:**
```bash
# Check if port 8080 is open
uv run gpumanager open-port --ip <manager-ip> --ports 8000,8080

# Check services
ssh <manager-ip>
cd /srv/shared
docker compose logs -f
```

**Admin login fails:**
- Check deployment logs for auto-generated credentials
- Or redeploy: `uv run gpumanager deploy --manager <manager-ip>`

### GPU Nodes

**Ollama not responding:**
```bash
# Test locally
ssh <gpu-node-ip>
curl localhost:11434

# Check logs
cd /srv/shared
docker compose logs ollama
```

**Models not loading:**
```bash
# Check if models are pulling
docker compose logs ollama | grep -i pull

# Manually pull a model
docker exec -it ollama ollama pull llama3.1:8b-instruct-q8_0
```

**Can't connect from WebUI:**
```bash
# Ensure port 11434 is open
uv run gpumanager open-port --ip <gpu-node-ip> --port 11434

# Test from manager node
ssh <manager-ip>
curl http://<gpu-node-ip>:11434
```

## Advanced

### Load Balancing

GPU Manager API handles intelligent load balancing:
- Automatically discovers and monitors all GPU nodes
- Routes requests to available GPUs with the required model
- Manages GPU lifecycle (pause/resume) based on demand
- OpenWebUI connects to a single endpoint (GPU Manager API)
- No manual configuration needed

### High Availability

For production:
- Deploy multiple manager nodes behind a load balancer
- Use shared storage for WebUI data
- Keep GPU nodes stateless

### Custom Models Path

Models are stored in Docker volumes on each GPU node:
- Path: `/root/.ollama` (inside container)
- Volume: `ollama_data`
- Persists across redeployments

## Security

- **Basic Auth:** All WebUI requests require authentication (via Caddy)
- **Firewall:** Only necessary ports opened in cloud firewall
- **Credentials:** Auto-generated secure passwords (check logs)
- **Private Network:** GPU nodes should be on private network when possible

## Support

For issues or questions:
- Check logs: `docker compose logs -f`
- Verify firewall rules are correct
- Ensure SSH access is configured
- Check GPU availability: `nvidia-smi`
